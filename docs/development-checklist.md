# 開発チェックリスト
この文書は [architecture.md](architecture.md)、[repository-structure.md](repository-structure.md)、[UX-policy.md](UX-policy.md)、[coding-rules.md](coding-rules.md) から導く短い統合チェックである。
## A. 構造
- [ ] 同じ仕様、状態、部品を複数箇所へ写さず、正本を一か所に保っている。
- [ ] ページ固有コードは入力取得、物理量変換、固有文言と課題だけを持つ。
- [ ] 複数ページで一緒に変わる処理は共通部品へ置いている。
- [ ] 位置と加速度など、見た目や操作が同じグラフはなるべく同じ描画部品を再利用している。
## B. 実装
- [ ] 画面上の成功、内部状態、チュートリアル判定が同じ状態源またはイベントへ接続している。
- [ ] 権限要求は利用者の開始操作を起点にしている。
- [ ] `visibilitychange` で計測を止め、リスナー、MediaStream、AudioContextを解放している。
- [ ] 復帰時に計測を自動再開していない。
- [ ] 実行時に外部CDNや外部ネットワークへ依存していない。
## C. UX
- [ ] 初見の利用者が30秒以内に、何を見るか、何を押すか、何が起きるかを理解できる。
- [ ] 物理現象と計測結果が主役で、設定や説明が前に出すぎていない。
- [ ] 状態文、ボタン、エラー文が現在状態と次の一手を示している。
- [ ] オンボーディングは最大3ステップで、最後は `#missionBtn` を指している。
- [ ] チュートリアルの表示上の成功と合格判定がずれていない。
## D. 配置と検証
- [ ] 編集は `src/` の正本へ行い、ルートHTMLを直接編集していない。
- [ ] `src/` の変更後に `python3 build.py` を実行し、生成物も更新している。
- [ ] `git diff --check` を通している。
- [ ] 変更に応じて `python3 -m unittest discover -s tests -v` と HTTPS 実機確認を実施している。
