# User Job：高校の物理基礎を受講する生徒

## この文書の立場

本書は、Bob Moesta 系の Jobs to Be Done を軸に、生徒がこのアプリを授業中にどう hire / fire するかを整理する。あわせて、Steve Krug の「考えさせるな」を、最初の理解負荷を下げる設計原理として扱う。

- customer は教員
- user は生徒
- 授業中に実際に hire / fire するのは生徒
- 2026年7月19日時点で、生徒本人の switch interview は未実施

## 主人公

学校端末を使い、位置・速度・加速度・時間の意味を、自分の体や身近な動きと結びつけて理解したい高校の物理基礎の生徒。

## Struggling Moment

板書や動画だけでは、グラフと現実の動きが頭の中でつながらない。しかも授業時間は短く、アプリで迷うと「自分だけ置いていかれる」と感じる。この瞬間に、生徒は説明より先に「自分で確かめられる体験」を必要とする。

## Job Statement

> 物理基礎の授業で、動きや音を自分で測って確かめたいとき、学校端末ですぐ計測を始め、予想と結果のズレをその場で見たい。そうすれば、公式を覚えるだけでなく、グラフの意味を自分の経験として理解できる。

## 望む進歩

- QRやURLからすぐ開ける
- 何を押せばよいか30秒以内に分かる
- 自分の動きとグラフがその場で結びつく
- 失敗しても自分でやり直せる
- 位置・速度・加速度を別物ではなく連続した理解として見られる
- 班の誰か1人だけでなく、自分も説明できる

## Four Forces

### Push

- 板書や口頭説明だけでは、グラフが自分事になりにくい
- 先生の演示だけでは、「自分で確かめた感覚」が弱い
- 手動計測では細かい変化やグラフの形が見えにくい

### Pull

- 自分の動きや音が、その場で数値とグラフに変わる
- インストール不要で授業の流れにそのまま入れる
- 別ページでも学んだ操作を使い回せる
- チュートリアルと状態文で、先生を待たずに進める
- 「考えさせるな」: 最初の30秒で、見る場所・押す場所・何が起きたかが直感で分かる

### Anxiety

- 権限ダイアログで止まるかもしれない
- 数値が跳ねると、壊れているように感じる
- 自分の班だけ動かず、恥ずかしい思いをするかもしれない
- 操作ばかり気にして、現象を見る時間が減るかもしれない

### Habit

- ノートを写して先生の説明を待つ方が安全
- 紙と手動ストップウォッチは分かりやすく失敗しにくい
- シミュレーションは見本としてきれいで早い
- 班の中で分かる人に任せる方が楽

## 競合分析

- `先生の演示・板書`: 迷わないが、自分で確かめた感覚は弱い
- `PhET`: 分かりやすいが、自分の実測ではない
- `phyphox`: 本格的だが、授業中には複雑に感じやすい
- `Vernier / PASCO`: 強力だが、器具やライセンスが前提になりやすい
- `Tracker`: 後から深く見られるが、その場のリアルタイム体験ではない
- `紙と手計測`: 安全だが、細かい変化や連続した見え方は弱い

## 採用タイムライン

1. `First Thought` 公式は分かるが、なぜそのグラフになるか実感できない
2. `Passive Looking` 先生からURLやQRを示され、「今日は自分で測るらしい」と知る
3. `Active Looking` 本当にすぐ始まるか、何を押すか、自分の班でも動くかを見る
4. `First Use` 最初の1回でグラフが出て、「自分の動きとつながった」と感じる
5. `Repeat Use` 別ページでも似た操作で進められ、「また使える」と判断する
6. `Fire` 迷う、止まる、意味が分からない体験が続くと離脱する

## 設計含意

1. 教員向けの売り文句より先に、生徒が30秒で始められるUIを守る
2. 最初の成功までの距離を、機能追加より優先して短くする
3. 「考えさせるな」を、説明を減らすことではなく認知負荷を下げることとして使う
4. 位置・速度・加速度を、別世界の画面ではなく連続した学習体験にする
5. 失敗時は原因説明より先に「次に押すこと」を示す
6. 班の誰か1人だけが分かる設計を避ける
7. 「観察」が主役で、「設定」が主役にならない画面にする

## Boundary Note

本書は仮説文書であり、次は確定していない。

- 生徒が最も強く欲しがるのが速度理解か加速度理解か
- どの競合をどの単元で最も強く代替するのか
- 生徒が「迷った」と感じる許容時間
- 教員が採用判断で最も重視する条件
