## Spec

### 決定事項

- 通常表示は `タップでグラフサイズ拡大` を主操作とする
- 一時停止中でも、拡大後にグラフ本体まで正しく再サイズされるようにする
- 通常表示では単指ドラッグも2本指ピンチも無効にし、拡大表示の中だけ操作を有効にする
- 拡大表示中の2本指ピンチは、時間軸だけではなく見え方全体として自然になるよう、縦軸も同時に縮尺変更する
- `pointer-events: none` な見出しを前提に、死んでいる `enableHeadToggle` 補助は削除する

### 却下案

- `enableHeadToggle` を残して見出しタップ補助を守る案
- 通常表示でも2本指ピンチで時間軸だけを変える案
- ピンチを完全に廃止して、拡大と値読みだけへ絞る案

## 作業ログ

- `src/lab-common.js` の `uplot-mount` 高さを inline style から CSS 変数へ変更し、`body.zoom` の `60vh` が勝つようにした
- zoom 切替直後に `scheduleZoomRefresh()` を予約し、一時停止中でもグラフ本体の再サイズが走るようにした
- 未拡大では単指ドラッグを無効にし、タップ判定をつぶさないため `TAP_SLOP_PX = 8` を導入した
- 2本指ピンチを `zoom中だけ` に制限し、2次元距離と `y scale` 更新で縦軸も同時に縮尺変更するようにした
- `enableHeadToggle` と、その前提テストを削除した
- `tests/test_lab_common.py` を増補し、拡大・ピンチ・再サイズの回帰を固定した
- `python3 build.py` を実行して生成物を更新した
- `python3 -m unittest discover -s tests -v` を実行し、`23 tests OK` を確認した
- `git diff --check` を実行し、問題なしを確認した

## Result

- `acceleration / gyroscope / magnetometer / location` の共通グラフで、タップ拡大の意味が `グラフサイズ拡大` として一貫した
- 一時停止中でも枠だけでなく、グラフ本体まで拡大されるようになった
- ピンチは拡大表示の中だけ有効になり、一般的な期待に近い `全体の見え方` の縮尺変更になった
- 死んでいた `enableHeadToggle` を削り、差分を少し整理できた

## Discussion

- docs の checklist 観点では、通常表示の主操作は `タップで拡大` に寄せる方が自然だった
- `横軸だけ変わるピンチ` は一般利用者の期待からずれやすく、`user-checklist` の `迷いにくい` を弱めるため採用しなかった
- 今回はピンチを残したが、将来さらに簡素化するなら `ピンチ自体を廃止` しても筋は通る
