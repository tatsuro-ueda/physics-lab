# アーキテクチャ設計
タブレット物理実験ラボを、学校端末で安定して使え、ページが増えても重複実装しない構造に保つ。本書は責務分担、実装、ビルド、検証の正本である。利用者体験は [UX-policy.md](UX-policy.md)、配置は [repository-structure.md](repository-structure.md)、実装判断の短い規約は [coding-rules.md](coding-rules.md) を参照する。

## 設計原則

- 同じ仕様、状態、部品は一か所だけを正本にする。
- ページ固有コードは、入力取得、物理量変換、ページ固有文言と課題だけを持つ。
- 画面上の成功とチュートリアルの合格判定は同じ状態またはイベントへ接続する。
- センサーとマイクは必要な間だけ確保し、画面を離れたら解放する。
- 実行時に外部ネットワークへ依存しない。
- `src/` の正本から、配布用の単一HTMLを再現可能に生成する。

## 構成

- `src/*.html`: ページ固有のHTML、入力購読、単位・符号補正、状態機械、ページ固有エラー文。
- `src/lab-common.js`: 3軸計測の保持、グラフ描画、タップ、拡大縮小、開始停止削除、チュートリアルへ公開する状態。
- `src/tutorial.js`: 体験チュートリアルのDOM、進捗、保存、`tick` / `event`。
- `src/tour.js`: 初回オンボーディングの起動、既読保存、driver.jsのラッパー。
- `src/orientation-guard.js`: 横向きタッチ端末への縦向き案内。
- `build.py`: `src/*.html` が参照するローカルJS・CSSをインライン化し、ルートHTMLを生成する。

## データフロー
- センサー型ページは `開始操作 -> 許可取得 -> 入力補正 -> lab.push() -> UI更新 -> tut.tick(e)` の順で流す。
- イベント型ページは `idle / armed / running / paused` をページ固有コードで持ち、表示更新と同時に `tut.event()` へ通知する。
- 表示だけ更新してチュートリアルへ通知しない実装は禁止する。
- チュートリアルが見る状態は仮値で返さない。

## 実行時ルール

- 権限要求は利用者のクリックまたはタップを起点にする。
- `visibilitychange` でページが隠れたら計測を停止し、リスナー、MediaStream、AudioContextを解放する。
- 復帰時に計測を自動再開しない。
- オンボーディング既読とチュートリアル進捗は、ページごとに一意な `localStorage` キーへ保存する。
- 計測データのリセットと学習進捗の保持は分ける。

## 検証規約

- `git diff --check`
- `python3 -m unittest discover -s tests -v`
- `python3 build.py`
- 共通部品の変更は利用する全ページへ反映されていることを確認する。
- センサー、マイク、権限変更は対象実機とHTTPSで手動確認する。

## 変更判断
- 複数ページで同じ変更が必要なら、先に共通部品へ置けないか検討する。
- 共通部品へ責務を足すときは、既存ページを壊さないAPI境界を先に決める。
- ページ固有の物理的意味を共通部品へ押し込みすぎない。
- ルートHTMLだけに存在する修正を作らない。
