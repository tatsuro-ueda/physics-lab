# 位置ページ docs整列 設計

日付: 2026-07-18
対象: `src/location.html`

## 目的

`location` ページを、`docs/UX-policy.md`、`docs/architecture.md`、`docs/customer-jobs.md` の方針へ合わせる。特に次を満たす。

- 初見の先生・生徒が30秒以内に、何を見るか、何を押すか、何が起きるかを理解できる。
- 初回ツアーは「場所案内」、チュートリアルは「体験」として役割を分ける。
- 速度・加速度も、場所の変化と結び付けて理解できるようにする。

## 現状の問題

- `location` は独自実装のままで、`tutorial.js` と `tour.js` を使う共通導線へ乗っていない。
- ヘッダーに `#missionBtn` がなく、他ページと学習導線がそろっていない。
- 初回オンボーディングがなく、初見利用者が何を見ればよいかを画面だけで把握しにくい。
- 速度・加速度タブ下部の地図がベクトル軌跡中心で、位置ベースの理解につながりにくい。
- ヒント文と状態表示が、現行のUX規約ほど明確に整理されていない。

## 採用する方式

### 方針

最小完全整列を採用する。

- GPS計測ロジック、時系列グラフ、タブ構成は `location` ページ固有コードとして残す。
- 学習導線と初回導線は、他ページと同じく `tutorial.js` と `tour.js` を使う。
- 速度・加速度タブ下部の地図は、独自の速度ベクトル地図や加速度ベクトル地図ではなく、位置軌跡 `track` を土台にした位置ベース表示へ置き換える。

### 採用理由

- `docs/UX-policy.md` の「30秒理解」「共通操作の再利用」「チュートリアル導線の統一」を満たせる。
- `docs/customer-jobs.md` の「教師が操作説明ではなく観察と解釈へ時間を使える」を助けられる。
- GPSページ固有の価値である「どこで何が起きたか」を、速度や加速度でも失わずに示せる。

## 画面構成

- ヘッダーは `戻る / ページ名 / 開始・停止 / リセット / チュートリアル` の並びにする。
- `#missionBtn` を追加し、チュートリアルへの入口を加速度ページと同じ位置へ置く。
- タブは現行の `位置 / 速度 / 加速度` を維持する。
- タブ直下へ `tutorialMount` を追加し、`tutorial.js` のパネルを差し込む。
- 数値カードは各タブの主役だけを上段に残す。
  - 位置: 緯度、経度、水平精度、移動距離
  - 速度: 速さ
  - 加速度: 加速度の大きさ
- x/y/z の時系列グラフは維持する。
- 画面下部の大きい地図は3タブとも「位置軌跡」を土台にする。
  - 位置: 通常の移動軌跡
  - 速度: 速い区間が分かる位置ベース表示
  - 加速度: 変化が大きい区間が分かる位置ベース表示
- ヒント文は通常時と拡大時で切り替える。

## 初回オンボーディング

`tour.js` を使い、初回訪問時だけ3ステップの場所案内を自動表示する。

- 保存キーは `location-onboarding-seen` とする。
- ステップは次の3つに固定する。
  1. 地図カード: ここで移動のようすを見る
  2. `#playBtn`: 押すと位置の記録が始まる
  3. `#missionBtn`: 押して使い方を体験してみてね
- 最終ステップは必ず `#missionBtn` を指す。
- 説明文は20文字程度を目安に短く保つ。

## 体験チュートリアル

`tutorial.js` を使い、位置タブ中心の必須3課題を提供する。

### 必須課題

1. `▶を押して記録を始めよう`
2. `少し歩いて、線や地図が動くのを見よう`
3. `地図で、どちらへ動いたか見てみよう`

### 判定方針

- 1は `playBtn` による開始で合格する。
- 2は `track` に点がたまり、移動距離がしきい値を超えたら自動合格する。
- 3は観察課題とし、`見えた！つぎへ` で進める。

### もっとやる

完了後の `もっとやる` は位置タブ内に限定する。

- グラフを拡大する
- 線をタップして値を読む

最初の導線では、速度・加速度タブまで課題を広げない。

## 地図表示の設計

### 共通方針

- 下部地図の基礎データは `track` のみを使う。
- `vtrack` と `atrack` の専用地図は廃止する。
- 利用者に「どこでその変化が起きたか」を場所で結び付けさせることを優先する。

### タブごとの見せ方

- 位置タブ: 現行どおり、開始点と現在位置が分かる移動軌跡を示す。
- 速度タブ: 同じ位置軌跡の上で、速い区間が目立つように色または線の強調を加える。
- 加速度タブ: 同じ位置軌跡の上で、速度変化が大きい区間が目立つように色または線の強調を加える。

強調方式は、既存描画コードへの組み込みが軽い方法を選ぶ。別の座標系や別地図を増やさない。

## 状態とライフサイクル

- 位置情報の取得は、利用者が開始を押した直後にだけ始める。
- 計測中は `#playBtn` を停止アイコンへ変える。
- もう一度押すと一時停止し、再開時は時間軸を連続させる。
- `visibilitychange` でページが隠れたら `watchPosition` を止める。
- 復帰時に自動再開しない。
- リセットは計測値、現在値、地図、マーカーを消す。
- リセットしてもチュートリアル進捗とオンボーディング既読は消さない。

## エラーと案内

少なくとも次の4種類を区別してページ内へ表示する。

- API非対応
- 権限拒否
- 取得失敗
- タイムアウト

文言は技術説明だけで終わらせず、次に確認すべき行動を短く添える。

## 実装境界

- 編集対象は `src/location.html` とする。
- 生成物の `location.html` は `python3 build.py` で更新する。
- `tutorial.js` と `tour.js` は正本を流用し、`location` のための分岐を最小限にとどめる。
- GPS計測、数値微分、位置軌跡生成、グラフ描画はページ固有責務として `src/location.html` に残す。

## 影響ファイル

- `src/location.html`
  - ヘッダーへ `#missionBtn` を追加
  - `tutorialMount` を追加
  - `driver.css`、`tutorial.js`、`tour.js` の導入
  - ツアー定義、チュートリアル定義の追加
  - 下部地図の描画モードを位置ベースへ変更
  - ヒント文、状態文、エラー文の整理
- `location.html`
  - `build.py` 実行結果として更新
- `tests/`
  - 必要なら `location` の導線存在確認を追加

## 検証

### 自動検証

1. `git diff --check`
2. `python3 -m unittest discover -s tests -v`
3. `python3 build.py`
4. ビルド後の `location.html` に `missionBtn`、`tutorialMount`、オンボーディング導線が反映されていること

### 手動確認

1. 初回だけツアーが自動で出る
2. 最終ステップが `#missionBtn` を指す
3. チュートリアル必須3課題が指示どおりに進む
4. 速度・加速度タブ下の地図が位置ベースで理解できる
5. タブ離脱で停止し、戻っても自動再開しない
6. 権限拒否、非対応、取得失敗、タイムアウトがページ内で区別表示される

GPS権限、実際の移動、`visibilitychange` の停止確認は、HTTPS上の実機確認を必須とする。

## 非対象

- `location` を `lab-common.js` 化する大規模共通化
- 速度・加速度タブを最初のチュートリアル必須課題へ含めること
- GPSデータの保存、提出、共有機能
- 他ページのUI改修
