# Customer Checklist
このチェックリストは [customer-journey.md](customer-journey.md) をもとに、教員視点の設計レビューで使う。
## 導入適合
- [ ] 既存授業のどの不満を解くかが一文で言える。
- [ ] customer が教員、user が生徒であることを混同していない。
- [ ] 価値が「高機能」より「授業が止まらない」「生徒が自走する」へ寄っている。
## 授業運用
- [ ] 生徒が30秒以内に始められる前提になっている。
- [ ] 教員の長い口頭説明や個別サポート依存を下げている。
- [ ] 権限差、端末差、失敗時の回復導線を授業リスクとして扱っている。
## 継続採用
- [ ] 別ページでも似た操作で再利用できる。
- [ ] 他教員へ30秒で説明できる。
- [ ] 専用器具や大きな予算がなくても価値が成立する。
- [ ] PhET、phyphox、演示授業より採用する理由と負ける条件が明確である。
