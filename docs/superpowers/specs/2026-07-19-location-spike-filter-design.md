# 位置ページ 単発スパイク除去 仕様

対象: `src/location.html`

## 目的

GPS の単発ぶれで、速度と加速度のグラフや下部地図に不自然なスパイクが出ることがある。
今回の目的は、歩行中の本物の変化はなるべく残したまま、単発の外れ値だけを落とすこと。

## 採用方針

- 変位グラフ (`x / y / z`) と位置タブの軌跡地図は生データの意味を保つため、そのまま残す。
- 速度 (`vx / vy / vz`) は、生の差分値を一度バッファし、3点中央値で中央サンプルを確定してから記録する。
- 加速度 (`ax / ay / az`) は、確定済みの速度から求め、生の差分値を同じく 3点中央値で中央サンプル確定してから記録する。
- 速度タブと加速度タブ下部の地図強調は、それぞれ確定済みの速度・加速度の絶対値を使う。

## 意味上の制約

- 3つのタブの上段グラフは、引き続き `変位 / 速度 / 加速度` の役割を保つ。
- 「位置ベース」にするのは下部地図の理解補助だけで、速度グラフや加速度グラフまで変位化しない。
- フィルタ対象は「単発スパイク」であり、連続する急な変化を全面的になめらかにすることは目的にしない。

## 実装方針

- `src/location.html` 内に小さな純粋関数を追加し、3要素の中央値を返せるようにする。
- 速度用と加速度用に、生値を短く保持するバッファを持つ。
- 3点そろった時点で中央サンプルだけを確定値として `SERIES` に push する。
- 確定が 1 サンプル遅れるため、速度・加速度の数値表示と下部地図強調も同じタイミングで更新する。
- リセット時は既存系列だけでなく、途中確定待ちのバッファも空にする。

## 期待する見え方

- その場で一瞬だけ GPS が飛んだとき、速度や加速度の単発ピークが出にくくなる。
- 本当に速く動き続けた区間は、速度・加速度の変化として残る。
- 位置タブの変位グラフと軌跡地図は、今回の変更前と同じ意味で読める。

## テスト方針

- `tests/` に 3点中央値の純粋関数テストを追加し、中央 1 点だけ跳ねた列が抑えられることを確認する。
- 既存の `tests/test_location_page.py` は、位置ページの導線と構造回帰を引き続き守る。
- `python3 -m unittest discover -s tests -v` と `python3 build.py` を通す。
