# User Journey：高校の物理基礎の生徒

## 前提

- user は生徒
- customer は教員
- この文書は「授業中に生徒が hire / fire する旅程」を扱う
- Bob Moesta 系の `struggling moment` と `four forces` を簡略化して使う
- Steve Krug の「考えさせるな」を、最初の理解負荷を下げる原理として使う

## 進歩

> 自分の動きや音をその場で迷わず測り、グラフの意味を自分の経験として理解したい。

## ジャーニー

| 段階 | 生徒の頭の中 | 競合 | 続ける条件 | 離脱条件 |
|---|---|---|---|---|
| 1. 授業導入 | 今日は何を見るのか | 先生の説明、板書 | 目的が一文で分かる | 何の実験か分からない |
| 2. 開始 | 何を押せばいいのか | 友達任せ、紙手順 | 見れば押す場所が分かる | URLや権限で止まる |
| 3. 最初の成功 | 本当に測れているか | 演示、PhET | 1回目で値かグラフが動く | 反応しない、何が起きたか分からない |
| 4. 観察 | この線は何を表すのか | 教科書の図、PhET | 動きと線が結びつく | 数字だけ見て終わる |
| 5. 比較 | 位置、速度、加速度はどう違うのか | Tracker、手計測 | 同じ流れで見比べられる | ページごとに別物に見える |
| 6. 班内説明 | 何が起きたか話せるか | 分かる友達1人に依存 | 用語と単位がそろっている | 操作役しか理解しない |
| 7. 振り返り | また使えば分かりそうか | ノート写し、先生まとめ | 再挑戦しやすい | 失敗の印象だけ残る |

## Struggling Moment

公式は覚えても、なぜそのグラフになるか実感できない。
さらにアプリで迷うと、自分だけ置いていかれる感じが強い。
この瞬間に生徒が欲しいのは長い説明ではなく、見る場所・押す場所・何が起きたかが直感で分かる体験である。

## 続けたくなる理由

1. 見る場所・押す場所・何が起きたかが直感で分かる
2. 最初の30秒で始められる
3. 自分の動きがすぐグラフになる
4. 間違えても先生を待たずにやり直せる
5. 別ページでも学んだ操作が使える

## やめたくなる理由

- 権限や端末差で止まる
- 数値が跳ねて信じられない
- 説明を読まないと操作や状態が分からない
- 操作ばかりで現象を見られない
- 友達だけ分かって、自分は分からないままになる

## 競合の見え方

- `先生の演示`: 迷わないが、自分で確かめた感じは弱い
- `PhET`: 分かりやすいが、自分の実測ではない
- `phyphox`: 本格的だが、授業中には複雑に感じやすい
- `Tracker`: 後から深く見られるが、その場の体験ではない
- `紙と手計測`: 安全だが、細かい変化は見えにくい

## 設計含意

1. 最初の成功までの距離を最短にする
2. 「考えさせるな」を、説明を減らすことではなく認知負荷を下げることとして使う
3. 失敗時は原因より次の一手を出す
4. 位置、速度、加速度を連続した体験として見せる
5. 班の誰か1人だけが理解する構図を避ける
6. 「観察」が主役で「設定」が主役にならない画面にする
