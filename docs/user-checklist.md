# User Checklist
このチェックリストは [user-journey.md](user-journey.md) をもとに、物理基礎の生徒視点の設計レビューで使う。
## 開始と最初の成功
- [ ] 今日は何を見るか、何を押すか、何が起きるかを30秒以内に理解できる。
- [ ] 見れば押す場所が分かり、説明を読まないと動けない状態になっていない。
- [ ] 最初の1回で値またはグラフが動き、「測れた」と実感できる。
## 理解の深まり
- [ ] 自分の動きとグラフが結びつき、現象の変化を読み取れる。
- [ ] 位置、速度、加速度を別物ではなく連続した体験として見比べられる。
- [ ] 設定より観察が主役で、どこを見れば理解が進むか迷いにくい。
## 班活動と継続
- [ ] 操作役だけが理解する構図を避け、班で予想と結果を話しやすい。
- [ ] 権限拒否、数値の跳ね、端末差が「自分だけ置いていかれる不安」を増やしにくい。
- [ ] 別ページでも同じ感覚で使え、再挑戦したくなる。
- [ ] 演示、PhET、phyphox、紙と手計測より「自分で確かめた感じ」が強い。
