# 位置ページ Safari クラッシュ軽減 仕様

対象: `src/location.html`

## 問題

iOS Safari で `location.html` を使っていると、画面ロックや復帰の前後をきっかけに Safari 自体が落ちることがある。
単なる「計測停止」ではなく、ブラウザが落ちる挙動として扱う。

## 根本原因の見立て

- 描画は `requestAnimationFrame(draw)` により常時ループしている。
- そのループ内で、全グラフ・全地図を毎フレーム描き直している。
- `drawGraph()` では毎回 `Math.min(...s.v)` / `Math.max(...s.v)` を全履歴配列へ使っている。
- 履歴が長くなるほど、Canvas 再描画量と配列展開コストが増え、Safari のメモリ・CPU負荷が高くなる。
- 画面ロックや復帰の前後は Safari がタブ状態を切り替えるため、重いページほど落ちやすい。

## 採用方針

- 常時描画ループをやめる。
- 描画は「新しいGPS点が来たとき」「タブ切り替え」「拡大縮小」「リセット」「開始/再開」など、必要時だけ行う。
- 連続イベントで無駄に多重描画しないよう、`requestAnimationFrame` は 1 回ぶんだけ予約する `scheduleDraw()` 方式にする。
- `Math.min(...s.v)` / `Math.max(...s.v)` のような配列展開はやめ、単一ループで最小値・最大値を集計する。

## 変えないこと

- 3つのタブの上段グラフは、引き続き `変位 / 速度 / 加速度` の意味を保つ。
- 速度・加速度の 3点中央値フィルタ方針は維持する。
- `visibilitychange` で隠れたら停止する仕様は維持する。
- チュートリアル、オンボーディング、位置ベースの下部地図の意味は変えない。

## 実装方針

- `draw()` は再帰ループをやめ、1回描画する関数にする。
- `scheduleDraw()` を追加し、未予約なら `requestAnimationFrame` を 1 回だけ積む。
- `startWatching()`、`onPos()`、タブ切り替え、グラフ拡大縮小、リセットから `scheduleDraw()` を呼ぶ。
- 系列の最小値・最大値を返す純粋関数を追加し、クリック時の当たり判定とグラフ描画の両方で再利用する。

## 期待する効果

- 長時間計測時の Safari 負荷を下げる。
- 画面ロックや復帰の前後でも、ブラウザクラッシュの起点になりにくくする。
- 見た目と学習導線は保ちつつ、内部の描画方式だけを軽くする。

## テスト方針

- `tests/test_location_page.py` に、常時ループ除去と `scheduleDraw()` 導入の回帰を追加する。
- 同テストで `Math.min(...s.v)` / `Math.max(...s.v)` が消えたことを確認する。
- `python3 -m unittest discover -s tests -v` と `python3 build.py` を通す。
