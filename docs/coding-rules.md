# Coding Rules

この文書は、実装時の判断をぶらさないための短い規約である。

## 原則

1. `SSOT`
   同じ仕様、状態、UI部品、判定条件を複数箇所へ写さない。正本は一か所に置く。
2. `DRY`
   複数ページで同じ処理が必要なら、先に共通部品へ置けないか検討する。
3. `共通部品優先`
   ページ固有コードは、入力取得、物理量変換、ページ固有の文言と課題だけを持つ。
4. `グラフ再利用`
   加速度ページで使うグラフ描画部品は、なるべくそのまま位置ページでも使う。見た目や操作が同じなら分岐より共通化を優先する。
5. `状態の一貫性`
   画面上の成功、内部状態、チュートリアルの合格判定は同じ状態源へ接続する。
6. `資源の節約`
   センサーとマイクは必要な間だけ確保し、画面を離れたら解放する。
7. `生成物直編集禁止`
   編集対象は `src/` の正本だけにし、ルートHTMLは `python3 build.py` の生成物として扱う。
8. `実行時オフライン`
   学校ネットワークを前提に、実行時に外部CDNへ依存しない。
9. `API境界を守る`
   共通部品へ新しい責務を足すときは、既存ページを壊さない入力と出力の境界を先に決める。
10. `物理の意味は押し込まない`
    ページ固有の物理的意味まで共通部品へ混ぜない。共通化するのは表示、操作、保存、進捗、状態管理である。
